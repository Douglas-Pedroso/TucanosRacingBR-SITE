🎨 CUSTOMIZAÇÕES RÁPIDAS

1. TROCAR LOGO:
   - Substitua /public/logo.png (512x512px ideal)
   - Formato: PNG com fundo transparente

2. ADICIONAR EVENTO:
   - Edite: src/pages/Eventos.jsx
   - Array: eventsData
   - Adicione um objeto com: id, name, track, date, category, description, participants

3. EDITAR RANKING:
   - Edite: src/pages/Ranking.jsx
   - Array: rankingData
   - Mude nome, pontos, vitórias, pódios

4. MUDAR CORES:
   - Edite: src/index.css ou arquivos .module.css
   - Verde: #22c55e → sua cor
   - Amarelo: #fbbf24 → sua cor
   - Dark: #0a0e27 → sua cor

🆘 TROUBLESHOOTING

Problema: "Aplicação em branco"
Solução: 
- Abra DevTools (F12)
- Vá em Console
- Verifique se há erros
- Se for erro de logo, verifique /public/logo.png

Problema: "Erro ao compilar"
Solução:
- npm run build
- Se der erro, veja a mensagem
- Verificar se todos os componentes têm import correto

Problema: "Dados desaparecem ao recarregar"
Solução:
- localStorage pode estar desabilitado
- DevTools → Application → Storage → localStorage
- Verifique se tem dados ali

Problema: "Logo não aparece"
Solução:
- Verifique /public/logo.png existe
- Tente recarregar (Ctrl+R ou Cmd+R)
- Limpe cache (Ctrl+Shift+Del)

💡 DICAS IMPORTANTES

1. Sempre testar localmente antes de fazer push:
   npm run dev → Testar → npm run build

2. Se tiver erro de build, não faça push:
   Corriga o erro localmente primeiro

3. GitHub Actions rodará automaticamente:
   - Verá na aba Actions do repositório
   - Se tiver ✅, tudo OK
   - Se tiver ❌, veja o log do erro

4. GitHub Pages leva 1-2 minutos para atualizar:
   - Faça push
   - Aguarde GitHub Actions terminar
   - Recarregue o site após 1-2 minutos

5. Para desenvolvimento contínuo:
   - Keep npm run dev rodando
   - Edite os arquivos
   - Vite fará HMR (atualização em tempo real)
